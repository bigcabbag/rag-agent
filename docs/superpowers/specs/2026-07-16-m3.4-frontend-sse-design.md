# M3.4 前端 SSE + 打字机 — Design

**日期**：2026-07-16  
**分支**：`feature/m3-react`  
**状态**：待实现（设计已口头批准）

## 目标

聊天框里看到 AI 逐 token 打出；流结束后挂 `sources`。可关回非流式 `POST /chat` 做对比。

## 已确认决策

| 项 | 选择 |
|----|------|
| 默认模式 | 流式开 |
| 非流式 | 开关关掉走现有 `postChat` |
| 中途取消 | 不做（无 Abort / 停止按钮）；流完才能再发 |
| 读流方式 | `fetch` + `ReadableStream` + 手写 SSE 解析（无新依赖） |
| 后端 | 不改（用现有 `POST /chat/stream`） |

## 后端契约（已有，只消费）

- `POST /chat/stream`，`Content-Type: application/json`（body 同 `ChatRequest`）
- 响应：`text/event-stream`
- 事件行：`data: {json}\n\n`
  - 增量：`{"token": "..."}`
  - 结束：`{"done": true, "model": "...", "sources": [...] | null}`
  - 错误：`{"error": "..."}`（仍可能是 200 + SSE 体内错误）

## 架构

```
ChatPanel
  ├─ streamOn=true  → postChatStream → onToken / onDone / onError
  └─ streamOn=false → postChat（M3.1 路径）
```

### 新增 `frontend/src/api/chatStream.ts`

- 导出 `postChatStream(body, handlers)`
- `fetch(`${API_BASE_URL}/chat/stream`, { method: "POST", ... })`
- HTTP `!ok`：解析 `detail` 后 throw（同 `chat.ts`）
- `response.body.getReader()` + `TextDecoder`；buffer 按 `\n\n` 拆事件
- 只处理以 `data: ` 开头的行；`JSON.parse` 后：
  - `token` → `handlers.onToken(token)`
  - `done` → `handlers.onDone({ model, sources })` 然后结束
  - `error` → throw / `onError`
- 复用 `ChatRequestBody` / `SourceChunk` 类型（从 `chat.ts` import）

### 改 `frontend/src/components/ChatPanel.tsx`

- 状态：`streamOn` 默认 `true`（UI 开关「流式」）
- 发送流程（流式）：
  1. 追加 user 消息
  2. 追加空 assistant 气泡（占位 id）
  3. `setLoading(true)`，禁用输入/发送
  4. `onToken`：按 id 追加 `content`
  5. `onDone`：写入 `sources` / `model`
  6. catch：该气泡标 `error` 或追加错误气泡
  7. `finally`：`loading=false`
- 非流式：保持现有 `postChat` 逻辑
- 样式：开关放在现有 `use_rag` 旁即可；最小 CSS

## 错误处理

| 情况 | 行为 |
|------|------|
| HTTP 4xx/5xx | throw，气泡显示错误文案 |
| SSE `{"error"}` | 同上 |
| 连接中断 / 无 `done` | throw「流式中断」类文案 |
| 向量库空 early_reply | 后端已发单 token + done；前端正常显示即可 |

## 不做（YAGNI）

- AbortController / 停止按钮
- Markdown 渲染
- 换 EventSource 或第三方 SSE 库
- 改后端事件格式

## 验收

1. 流式开：提问 → 气泡逐字增长 → 结束有 sources（RAG 且有库时）
2. 流式关：仍走 `/chat`，一次出全文
3. 流式中禁用重复发送
4. 人为断后端或错误时，UI 有错误提示
