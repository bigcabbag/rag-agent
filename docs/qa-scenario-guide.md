# 场景面试题规范（面经驱动）

> 从 M3 起，**每步协作的面试题必须是场景题**，不再是纯概念题。  
> 旧版概念题保留在 `qa-m*.md`，复习时可对照；**新题写入对应 milestone 的「场景题」章节**。

---

## 场景题 vs 概念题

| | 概念题（旧） | 场景题（新，默认） |
|---|-------------|-------------------|
| 问法 | 「CORS 是什么？」 | 「前端 5173 调 8000 报 CORS 错误，Swagger 正常，你怎么查？」 |
| 考察 | 背定义 | 排查思路 +  trade-off + 是否做过 |
| 答案 | 一段解释 | **现象 → 原因 → 步骤 → 更好方案** |

---

## 标准答案结构（STAR-Engineering）

每道场景题参考答案必须包含：

1. **现象**：用户/线上看到什么
2. **根因**：哪一层出了问题（对应代码/架构）
3. **排查/解决步骤**：先做什么、后做什么（可结合本项目）
4. **更好方案**（面经爱追问）：短期 fix vs 长期优化
5. **（可选）本项目关联**：指到具体文件或 milestone

## 面经场景题模板库（按类型）

### A. 量化与评估（腾讯/快手高频）

**题：** 你做了三个月 RAG，召回率多少？怎么量的？

**参考答：**
- **现象**：面试官质疑 Demo 没有指标。
- **做法**：建 `eval/questions.json` 20 题，每题标注期望来源文档；跑 `eval/run_eval.py` 算 **Recall@3**（正确 chunk 是否出现在 Top-3）。
- **基线**：M2 纯向量约 XX%；M4.3 加 BM25 混合后提升 XX%（有数字才能讲）。
- **更好方案**：上线后用 RAGAS 的 context_recall + faithfulness；生产采样 bad case 回流 eval 集。
- **本项目**：M4.1 实现，见 `docs/SCENARIO.md`。

---

### B. 回答不准 — 四层排查（腾讯云/派聪明）

**题：** 用户说「AI 答错了」，你从哪开始查？

**参考答：**

| 层 | 查什么 | 本项目 |
|----|--------|--------|
| 1. Query | 问题是否太模糊？ | 加 query 改写（M4.2 LangGraph） |
| 2. Retrieve | Top-K 片段是否相关？ | 看 `sources` 字段；Chroma 是否空 |
| 3. Chunk | 切块是否切断语义？ | `loader.py` chunk_size=500 |
| 4. Generate | 上下文对了但 LLM 仍胡说？ | 加强 `RAG_SYSTEM_PROMPT` 拒答 |

**更好方案**：先分 Retrieve vs Generate（看 gold chunk 在不在 Top-K），再动刀；不要一上来换模型。

---

### C. Retrieve vs Generate 诊断（RAGAS/Braintrust）

**题：** 怎么判断是检索问题还是生成问题？

**参考答：**
- **Retrieve 问题**：正确文档块**不在** Top-K → 查 embedding、切块、是否缺混合检索、索引是否过期。
- **Generate 问题**：正确块**在** Top-K 但答案错/瞎编 → 查 prompt grounding、temperature、是否缺 citation 约束。
- **工具**：离线用 RAGAS；手工把 Top-3 片段读一遍最快。
- **本项目**：M4.1 eval 脚本输出 retrieve 命中；M4.2 CRAG 评分低则改写 query 重检。

---

### D. 工程故障（真实开发会遇到）

**题：** 浏览器前端调 `/health` 失败，Swagger 可以，怎么办？

**参考答：**
- **现象**：仅浏览器失败，Postman/Swagger 正常。
- **根因**：几乎总是 **CORS** 或 URL 写错（5173 vs 8000）。
- **步骤**：F12 Network 看 preflight；确认 `main.py` `allow_origins` 含 `http://127.0.0.1:5173`；确认 `config.ts` API_BASE_URL。
- **更好方案**：生产同域 Nginx 反代，开发期 CORS，生产不需要。

---

**题：** SSE 流式到一半断了，前端怎么处理？

**参考答：**
- **现象**：打字机停在一半。
- **步骤**：保留已显示文本；提示用户；提供重试；可选自动重试 1 次。
- **更好方案**：后端 heartbeat；前端超时 timer；M3.4 实现时加 error state。
- **本项目**：M3.4 验收标准之一。

---

**题：** 上传 50MB PDF 到一半失败，怎么设计？

**参考答：**
- **现象**：大文件上传超时/中断。
- **短期**：前端限制大小 + 明确错误提示；后端异步处理返回 task_id。
- **更好方案**：分片上传 + Redis Bitmap 记录进度（腾讯 QQ 面经）；Celery 异步 embed。
- **本项目**：M2 同步上传够用；M5 README 写已知限制和扩展方向。

---

### E. 场景设计（快手/美团）

**题：** 研发团队文档从 10 篇涨到 10 万篇，你怎么改架构？

**参考答：**
- Chroma 换 **Qdrant/Milvus**；按 `source`/部门 metadata 过滤；增量索引；异步 ingest 队列。
- 检索：混合检索 + Rerank；热文档缓存。
- **不必一次全做**：简历写「当前 Chroma 适用 <10k 块；扩展方案已设计」。

---

**题：** 文档更新了，怎么不停服更新向量库？

**参考答：**
- 按 `source` 删旧向量再写入（本项目 `store.py` 已实现同名文件覆盖）。
- **更好方案**：双索引切换（blue-green）；后台异步 re-embed；版本号 metadata。
- **本项目**：`index_chunks` 先 delete 再 add，可口述为「单文件热更新」。

---

### F. Trade-off（美团 Keeta：RAG vs Agent）

**题：** 智能客服里，什么问题用 RAG，什么用 Agent/Skill？

**参考答：**
- **RAG 适合**：政策、FAQ、文档类——答案在知识库里（优惠券规则、部署步骤）。
- **Agent/Skill 适合**：要调 API、多步流程（查订单、退款状态）。
- **本项目**：DevKit 纯文档问答 → RAG 足够；M4.2 加 LangGraph 是为了**检索质量**，不是替代业务 API。

## 每步协作出题规则

完成 Mx.y 后，出 **2～4 道场景题**，必须：

- [ ] 与本步刚写的代码/验收相关
- [ ] 用「线上/用户/面试官」口吻提问
- [ ] 答案含「更好方案」
- [ ] 写入 `docs/qa-mx.md` 的 **「场景题」** 章节（与概念题分开）

### 示例：M3.0 场景题

**Q：** 同事说「前端连不上后端」，你第一次会打开哪三个地方？

**A：**
1. 浏览器 F12 → Network 看 CORS 还是 connection refused
2. 终端确认 `uv run python main.py` 是否在 8000 监听
3. `frontend/src/config.ts` 的 API_BASE_URL 是否指向 8000

**更好方案：** 加 `/health` 连通性自检页（我们 M3.0 已做）。
